فاز ۱: فونداسیون، دامنه و ذخیره‌سازی

Phase 1: Foundation, Domain & Storage

۱. خرید دامنه و مقایسه

برای برند isatis.pro، پلتفرم Porkbun انتخاب شد. دلیل: قیمت شفاف تمدید و WHOIS Privacy رایگان. در مقابل Namecheap که هزینه‌های پنهان دارد.

۲. تنظیمات Cloudflare

تمام رکوردهای DNS به کلودفلر منتقل شد تا از لایه امنیتی WAF استفاده شود.

مراحل:
۱. تغییر Nameservers در پنل Porkbun به آدرس‌های ارائه شده توسط کلودفلر.
۲. فعال‌سازی پروکسی (ابر نارنجی) برای رکوردهای اصلی سایت.

۳. فضای ذخیره‌سازی R2

انتخاب R2 به دلیل هزینه صفر خروجی (Egress Fee). تنظیم ساب‌دامین cdn.isatis.pro برای دسترسی مستقیم به فایل‌ها.

1. Domain Acquisition

Porkbun was chosen for isatis.pro due to transparent renewal pricing and free WHOIS privacy, unlike competitors with hidden fees.

2. Cloudflare Setup

DNS records were migrated to Cloudflare to utilize its WAF security layer.

Steps:
1. Change Nameservers in Porkbun to the ones provided by Cloudflare.
2. Enable Proxy (Orange Cloud) for main website records.

3. Cloudflare R2 Storage

Implemented R2 for its Zero Egress Fees. Configured cdn.isatis.pro as a custom domain for direct asset access.

فاز ۲: سرور، سیستم‌عامل و Coolify

Phase 2: Server, OS & Coolify

۱. انتخاب سرور هتزنر

استفاده از سرور ابری هتزنر (Hetzner Cloud) با سیستم‌عامل Ubuntu 24.04. دلیل: پایداری بالا در اروپا.

۲. مهاجرت از CloudPanel به Coolify

تجربه: کلودپنل برای PHP عالی است اما برای Next.js و داکر محدودیت داشت. کولفای جایگزین Vercel شد.

۳. نصب Coolify

curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash

۴. رفع باگ IPv6 (بسیار مهم)

شبکه IPv6 هتزنر باعث کرش کردن داکر می‌شود. برای حل آن:

nano /etc/docker/daemon.json
# Add: "ipv6": false

1. Hetzner Server Choice

Hetzner Cloud with Ubuntu 24.04 LTS was selected for its performance and European stability.

2. CloudPanel to Coolify Migration

Experience: CloudPanel is great for PHP but limited for Docker. Coolify was chosen as an open-source Vercel alternative.

3. Coolify Installation

curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash

4. Critical IPv6 Fix

Hetzner's IPv6 network causes Traefik/Docker crashes. Fix:

nano /etc/docker/daemon.json
# Add: "ipv6": false

فاز ۳: استقرار و پایگاه داده

Phase 3: Deployment & Database

۱. حل تفاوت پلتفرم (Mac vs Linux)

چون در مک کد می‌زنید، نصب پکیج‌ها در لینوکس با خطا مواجه می‌شود. راهکار در Coolify:

npm install --force --os=linux --cpu=x64

۲. دستورات بیلد پریزما

برای اطمینان از ساخته شدن جداول قبل از بالا آمدن سایت:

npx prisma generate && npx prisma db push && npm run build

۳. انتقال دیتابیس لوکال به سرور (Docker CP)

مرحله اول: ارسال به ریشه سرور با SCP. مرحله دوم: کپی به داخل کانتینر:

docker cp /root/dev.db :/app/prisma/dev.db

1. Platform Mismatch Fix (Mac vs Linux)

Since development is on Mac (ARM), use this in Coolify build settings to avoid EBADPLATFORM errors:

npm install --force --os=linux --cpu=x64

2. Prisma Build Pipeline

Ensure DB schema is pushed before Next.js build starts:

npx prisma generate && npx prisma db push && npm run build

3. Database Injection (Docker CP)

Step 1: SCP file to server. Step 2: Copy into the isolated container:

docker cp /root/dev.db :/app/prisma/dev.db

فاز ۴: پایداری، مانیتورینگ و نگهداری

Phase 4: Stability, Monitoring & Maintenance

۱. ساخت حافظه Swap (رم مجازی)

جلوگیری از کرش کردن سرور ۲ گیگابایتی در زمان بیلد سنگین:

fallocate -l 4G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

۲. مانیتورینگ دوگانه

Uptime Kuma: مانیتورینگ سایت از بیرون (نصب در Coolify).
Netdata: مانیتورینگ سخت‌افزار از داخل (نصب در لینوکس).

۳. هشدار تلگرام

اتصال Bot Token و Chat ID به هر دو برنامه برای دریافت نوتیفیکیشن روی گوشی.

۴. تحلیل دیسک و آپدیت

چک کردن فضای هارد با df -h و آپدیت پکیج‌ها با apt upgrade.

1. Creating 4GB Swap (Virtual RAM)

Prevents OOM crashes during heavy Next.js/Dart builds on 2GB RAM:

fallocate -l 4G /swapfile
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

2. Dual Monitoring

Uptime Kuma: External availability checks.
Netdata: Internal hardware health metrics.

3. Telegram Bot Alerts

Integrated Bot Token and Chat ID to receive instant alerts for downtime or high RAM usage.

4. Maintenance

Regular OS updates and disk cleanup using docker system df.
